# 文档一致性检查功能使用指南

## 功能概述

文档一致性检查功能可以自动检测修改文档后需要同步的关联文档，帮助维护设定文档之间的一致性。

## 工作原理

### 1. 文档关联关系

系统预定义了文档之间的关联关系：

- **主角成长路线图.md** ↔ 主要角色、功法术法、法宝图鉴、历史时间线
- **主要角色（粗略）.md** ↔ 成长路线、功法术法、法宝图鉴、势力宗门
- **功法与术法大全.md** ↔ 主要角色、成长路线、势力宗门
- **法宝图鉴.md** ↔ 主要角色、成长路线
- **世界之谜和答案.md** ↔ 历史时间线、成长路线
- **历史世界时间线.md** ↔ 世界之谜、主要角色

### 2. 实体提取

系统会自动从文档中提取以下实体：

- **角色名称**：中文名（2-4个字）
- **功法名称**：【】包裹，以"诀、法、术、功、拳、剑、阵"结尾
- **法宝名称**：【】包裹，以"剑、刀、枪、盾、符、珠、玉"等结尾
- **境界名称**：铁骨境、铜血境、摧城境等
- **地点名称**：中土神洲、东临剑洲、洗剑池等

### 3. 冲突检测

系统会检测以下类型的冲突：

- **境界不一致**：同一角色在不同文档中的境界描述不同（高优先级）
- **功法引用**：功法在多个文档中被提及，需要检查描述一致性
- **法宝引用**：法宝在多个文档中被提及，需要检查归属和品阶一致性

## 使用方法

### 方法1：通过MCP Server

```javascript
// 调用 check_consistency 工具
{
  "name": "check_consistency",
  "arguments": {
    "modifiedFile": "主角成长路线图.md",
    "modifiedContent": "文件的完整内容..."
  }
}
```

### 方法2：通过测试脚本

```bash
node test-consistency.js
```

## 工作流程建议

### 修改文档时的标准流程

1. **修改文档**：编辑你需要修改的设定文档
2. **运行一致性检查**：使用 `check_consistency` 工具检查关联文档
3. **查看报告**：查看哪些文档需要同步，优先处理高优先级的冲突
4. **手动同步**：根据建议，手动更新相关文档
5. **再次检查**：更新后再次运行一致性检查，确保没有遗漏

### 示例：修改主角成长路线后的同步流程

假设你在"主角成长路线图.md"中添加了：
- 宁观在摧城境获得了新法宝【雷击神木碎片】
- 裴青冥突破到筑鼎境

一致性检查会提示你：
1. 需要在"主要角色（粗略）.md"中更新裴青冥的境界
2. 需要在"法宝图鉴.md"中添加【雷击神木碎片】的条目
3. 需要在"主要角色（粗略）.md"中更新宁观的关键物品列表

## 优先级说明

- **优先级 10+**：境界不一致等严重冲突，必须立即处理
- **优先级 5-9**：功法、法宝引用，建议尽快检查
- **优先级 1-4**：一般性引用，可以稍后处理

## 限制和注意事项

1. **实体提取不完美**：系统可能会将一些常见词汇误识别为角色名，已经过滤了大部分常见词
2. **需要手动同步**：系统只提供建议，不会自动修改文档
3. **关联关系固定**：如果添加新的设定文件，需要在 `consistencyAgent.js` 中更新关联关系
4. **境界检测有限**：只能检测角色名称附近50个字符内的境界描述

## 未来改进方向

1. **自动同步**：提供一键同步功能，自动更新关联文档
2. **更智能的实体提取**：使用更复杂的NLP技术提取实体
3. **时间线一致性检查**：检测时间线上的矛盾
4. **可视化关联图**：展示文档之间的关联关系图
5. **版本对比**：对比文档的不同版本，追踪变更历史

### 核心方法

- `checkConsistency(modifiedFile, modifiedContent)`: 检查一致性
- `extractEntities(content)`: 提取实体
- `detectConflicts(sourceFile, sourceEntities, targetFile, targetContent)`: 检测冲突
- `generateSyncReport(consistencyResult)`: 生成报告

## 常见问题

**Q: 为什么有些角色名没有被识别？**
A: 系统只识别2-4个字的中文名，且会过滤常见词汇。如果角色名太短或太长，可能无法识别。

**Q: 如何添加新的文档关联关系？**
A: 编辑 `src/agents/consistencyAgent.js` 中的 `documentRelations` 对象。

**Q: 一致性检查会修改我的文档吗？**
A: 不会。系统只提供建议，所有修改都需要手动完成。

**Q: 如何提高检测准确率？**
A: 使用【】包裹功法和法宝名称，保持命名规范，避免使用过于常见的词汇作为角色名。
