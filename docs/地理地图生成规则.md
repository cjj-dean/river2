# 地理地图生成规则（当前版本）

## 数据源

- 脚本：`tools/geo_map_tool.py`
- 输入目录：`settings/卡片/地理/**/*.md`
- 输出：
  - 网页：`/`
  - 数据接口：`/api/geo-data`

---

## 坐标规则（核心变更）

- `[[相对坐标]]` 使用 **相对于父级中心点的百分比坐标**。
- 格式固定：`[x%,y%]`
  - `x%`：向右为正，向左为负
  - `y%`：向上为正，向下为负
- 不再使用卡片中的绝对坐标数值作为标准写法。
- 若读到旧坐标（如 `[120,30]`），normalize 会自动转换为百分比后写回。

---

## 半径规则（新增）

- 每张地理卡片新增字段：`[[相对半径]]`
- 定义：`当前卡片半径 / 父卡片半径`
- 顶层地理（父级是扶摇界）以“世界半径”作为父半径基准。
- normalize 会自动补齐该字段并回写到卡片。

---

## 边界规则（核心变更）

- 每个区域边界统一为 **圆形**。
- 数据层会输出：
  - `radius` / `boundaryRadius`
  - `boundary`（由圆周点采样得到的多边形，用于前端渲染）
- 不再使用不规则行政边界。

---

## normalize 做了什么

执行 `python3 tools/geo_map_tool.py normalize` 后：

- 扫描全部地理卡片
- 解析/推断基础字段（地理类型、父级、繁荣度等）
- 计算相对坐标与相对半径
- 回写标准字段（包含 `[[相对坐标]]` 与 `[[相对半径]]`）
- 旧绝对坐标会迁移为百分比坐标

---

## validate 校验项

执行 `python3 tools/geo_map_tool.py validate`：

- 校验每张卡片存在 `## 基本信息`
- 校验关键字段存在：
  - `地理类型`
  - `行政级别`
  - `所属父级`
  - `相对坐标`
  - `相对半径`
  - `繁荣度`
- 校验 `相对坐标` 格式必须是 `[x%,y%]`
- 校验 `相对半径` 为正数
- 校验子圆完全在父圆内

---

## 运行命令

```bash
python3 tools/geo_map_tool.py normalize
python3 tools/geo_map_tool.py validate
python3 tools/geo_map_tool.py serve --port 4177
```

访问地址：

- `http://localhost:4177/`
