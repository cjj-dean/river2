# 使用指南

## 快速开始

### 1. 准备工作

1. 确保已安装 Node.js (v18 或更高版本)
2. 获取 Anthropic API Key：https://console.anthropic.com/
3. 克隆或下载本项目

### 2. 安装依赖

```bash
cd novel-setting-editor
npm install
```

### 3. 配置环境变量

复制 `.env.example` 为 `.env`：

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入你的 API Key：

```
ANTHROPIC_API_KEY=sk-ant-xxxxx
```

### 4. 准备设定文件

将你的小说设定 Markdown 文件放入 `settings/` 目录。

已提供三个示例文件：
- `世界观.md`
- `流派和等级体系.md`
- `历史事件时间线.md`

你可以根据需要添加更多文件，如：
- `地图与势力分布.md`
- `人物关系与势力架构.md`
- `货币体系.md`
- `金手指.md`

### 5. 在 Claude Code 中配置

找到 Claude Code 的配置文件（通常在 `~/.config/claude-code/` 或类似位置），添加：

```json
{
  "mcpServers": {
    "novel-setting-editor": {
      "command": "node",
      "args": ["D:/claude/novel-setting-editor/src/index.js"],
      "env": {
        "ANTHROPIC_API_KEY": "sk-ant-xxxxx"
      }
    }
  }
}
```

**注意**：将路径替换为你的实际项目路径。

### 6. 重启 Claude Code

配置完成后，重启 Claude Code 使配置生效。

## 使用示例

### 示例 1：添加新势力

在 Claude Code 中输入：

```
我想在设定里增加一个专门猎杀主角所在流派的暗杀组织。
```

系统会：
1. 读取相关设定文件（世界观、流派体系、地图等）
2. 生成详细的组织设定草案
3. 通过三个审查官进行审查：
   - 战力审查官：检查组织的战力设定是否合理
   - 经济审查官：检查组织的资源来源是否合理
   - 时间线审查官：检查组织的历史是否与时间线冲突
4. 汇总审查结果，给出修改建议
5. 将最终设定写入相关文件

### 示例 2：添加新功法

```
添加一个适合暗杀的隐身功法，要求能够欺骗神识。
```

系统会根据世界观中"神识可以锁定空间"的设定，生成合理的功法描述。

### 示例 3：添加历史事件

```
在黑暗动乱期间，添加一个关于魔道崛起的重要事件。
```

系统会检查时间线，确保新事件不与现有历史冲突。

## 工作流程详解

### 阶段 1：提议（Proposal）

**工具**: `propose_setting`

Router Agent 会：
1. 分析你的需求
2. 选择相关的设定文件
3. 读取这些文件的内容
4. 基于现有设定生成新的设定草案

### 阶段 2：审查（Review）

**工具**: `review_setting`

三个专业审查官并行工作：

#### 战力审查官
- 检查力量等级是否符合现有体系
- 检查能力是否违反世界观法则
- 检查战力对比是否合理

#### 资源/经济审查官
- 检查资源分布是否与地理冲突
- 检查经济规模是否合理
- 检查货币体系一致性

#### 考据与时间线审查官
- 检查时间点是否冲突
- 检查历史渊源是否合理
- 检查因果关系

### 阶段 3：整合（Commit）

**工具**: `commit_setting`

如果审查通过或修改完成：
1. 确定需要更新的文件
2. 将新设定追加到相应文件
3. 返回操作结果

## 高级用法

### 自定义审查 Agent

你可以在 `src/agents/reviewCoordinator.js` 中添加新的审查官：

```javascript
{
  name: '魔法体系审查官',
  role: 'magic_reviewer',
  files: ['魔法体系.md', '世界观.md'],
  prompt: `你是魔法体系专家...`,
}
```

### 调整文件选择策略

在 `src/agents/routerAgent.js` 的 `selectRelevantFiles` 方法中，可以自定义文件选择逻辑。

### 批量处理

你可以编写脚本批量处理多个设定请求。

## 常见问题

### Q: API 调用失败怎么办？

A: 检查：
1. API Key 是否正确
2. 网络连接是否正常
3. API 配额是否充足

### Q: 审查结果不准确？

A: 可以：
1. 完善设定文件的内容
2. 调整审查官的 prompt
3. 增加更多相关的设定文件

### Q: 如何回滚错误的设定？

A: 设定文件是纯文本 Markdown，可以：
1. 使用 Git 版本控制
2. 手动编辑文件删除错误内容

### Q: 能否支持其他 LLM？

A: 可以修改 Agent 代码，替换 Anthropic SDK 为其他 LLM 的 SDK。

## 最佳实践

1. **使用版本控制**：将 `settings/` 目录纳入 Git 管理
2. **定期备份**：重要设定文件要定期备份
3. **结构化组织**：使用清晰的 Markdown 标题层级
4. **详细描述**：设定文件越详细，AI 生成的内容越准确
5. **人工审核**：AI 生成的内容仍需人工最终审核

## 性能优化

- 大型设定库可能导致 API 调用较慢
- 可以考虑实现缓存机制
- 可以使用向量数据库优化文件检索
