# Skill 目录总说明

## 一、定位

本目录用于存放《扶摇界》项目的专项 Skill 文件。

这些 Skill 的作用不是替代正式设定文件，而是：

1. 在新增设定时提供稳定的生成框架。
2. 在补全设定时提供统一的质量标准。
3. 在不同类型任务之间保持风格、逻辑和命名一致。

简单说：

- 正式 `.md` 设定文件负责“世界里已经确定了什么”。
- `skill` 文件负责“以后该怎么继续生成，才不跑偏”。

## 二、使用总原则

1. 若 `skill` 与正式设定文件冲突，以正式设定文件为准。
2. 若总纲 Skill 与专项 Skill 冲突，优先看专项 Skill 的任务边界。
3. 若用户要求是在现有世界观基础上补全，默认补全而不是推翻。
4. `小说基调和文风skill.md` 主要约束正文、剧情化文案和示例表达，不应过度限制设定文档中的工作术语。

## 三、Skill 分层

当前 Skill 分为三层：

### 1. 总纲层

- `小说设定30条原则skill.md`

作用：

- 负责世界观总原则校准。
- 用于判断一个新设定是否足够稳、够厚、够能支撑长篇。

### 2. 风格校准层

- `小说命名30条原则skill.md`
- `小说基调和文风skill.md`

作用：

- 一个负责术语和专有名词。
- 一个负责正文气质、角色表达和场景文风。

### 3. 专项生成层

- `世界起源设定skill.md`
- `世界历史设定skill.md`
- `能量体系设定skill.md`
- `流派和等级体系设定skill.md`
- `势力体系设定skill.md`
- `地理体系设定skill.md`
- `货币体系设定skill.md`
- `金手指体系设定skill.md`
- `荣誉体系设定skill.md`
- `人物设定skill.md`
- `故事主线设定skill.md`

作用：

- 面向具体设定项生成或补全内容。

## 四、推荐调用顺序

如果要从零搭一个长篇世界，建议按以下顺序推进：

1. `小说设定30条原则skill.md`
2. `小说基调和文风skill.md`
3. `世界起源设定skill.md`
4. `世界历史设定skill.md`
5. `能量体系设定skill.md`
6. `流派和等级体系设定skill.md`
7. `物种体系` 对应正式设定或后续专项 Skill
8. `势力体系设定skill.md`
9. `地理体系设定skill.md`
10. `货币体系设定skill.md`
11. `金手指体系设定skill.md`
12. `荣誉体系设定skill.md`
13. `人物设定skill.md`
14. `故事主线设定skill.md`
15. `小说命名30条原则skill.md` 用于最后统一术语和命名校准

如果不是从零开始，而是在现有项目中补全，建议按“缺什么补什么”的方式调用，但仍要先参考：

1. `小说设定30条原则skill.md`
2. `小说基调和文风skill.md`
3. 对应专项 Skill
4. `小说命名30条原则skill.md` 做最终命名校准

## 六、几个关键依赖关系

### 1. 起源 -> 历史 -> 现世

- `世界起源设定skill.md` 决定根问题。
- `世界历史设定skill.md` 决定这些问题如何一路滚到现在。

### 2. 能量 -> 等级 -> 势力

- `能量体系设定skill.md` 决定力量来源和代价。
- `流派和等级体系设定skill.md` 决定修行者怎么成长。
- `势力体系设定skill.md` 决定谁垄断这些力量。

### 3. 地理 -> 货币 -> 势力分布

- `地理体系设定skill.md` 决定资源在哪里。
- `货币体系设定skill.md` 决定资源如何流动。
- `势力体系设定skill.md` 决定谁控制流动。

### 4. 金手指 -> 人物 -> 主线

- `金手指体系设定skill.md` 决定主角或关键人物的异常变量。
- `人物设定skill.md` 决定变量落在谁身上。
- `故事主线设定skill.md` 决定这个变量如何撬动天下局势。

### 5. 荣誉 -> 势力 -> 人物上升路径

- `荣誉体系设定skill.md` 决定功劳如何转化成资格。
- `势力体系设定skill.md` 决定谁有权授予荣誉。
- `人物设定skill.md` 决定角色如何借此改变命运。

## 七、维护建议

1. 新增 Skill 时，尽量沿用当前统一模板：
   - `Role`
   - `Description`
   - `References`
   - `Rules`
   - `Required Output`
   - `Constraints`
   - `Startup`
2. 新增 Skill 前，先判断它属于总纲层、风格层还是专项层。
3. 若某个正式设定文件已经稳定，Skill 应以“补全与校准”为主，而不是反复重构。
4. 若多个 Skill 都会处理同一内容，必须在 `References` 中写清谁是上游、谁是下游。

## 八、一句话总结

`skill` 目录不是设定仓库，而是《扶摇界》后续持续生成设定时的“操作手册”。它的价值不在于替你写世界，而在于让你每次继续写世界时，都还能保持同一个世界。
